# Source Boundary

Phase 0 does not create a protocol implementation project.

- Implementation language remains Open in `DEC-DEV-001`.
- If an approved .NET SDK is used in a future phase, C# Console plus standard library should be reviewed first.
- If .NET is not approved and Python is available, Python standard library should be reviewed.
- External dependency introduction requires separate approval.
- Future implementation should separate Protocol, Transport, Scenario, Fault Injection, Trace, and Oracle responsibilities.
